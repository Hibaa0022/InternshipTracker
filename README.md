## InternshipTracker (Frontend Only)
A simple frontend-only internship tracking web app that helps users organize and monitor internship applications in one place.

**Repository:** Hibaa0022/InternshiptTracker

**Main file:** InternshipTracker.htm

## 📌 Overview

**InternshipTracker** is a lightweight web interface for tracking internship opportunities and application progress.
It is built as a **static frontend project (HTML/CSS/JavaScript)** and does not include a backend server or database.

This project is useful for students and job seekers who want a clean and easy way to keep application records organized.

## ✨ Features (Frontend)
Depending on your current UI implementation, this app can support:
* Add internship application entries.
* Track company, role, and application status.
* Update progress stages (e.g., Applied, Interview, Offer, Rejected)
* Organize entries in a simple visual layout.
* Manage data in-browser (if local storage is used).

## 🧱 Tech Stack
This repository is currently a frontend-only static app:
* **HTML** (structure)
* **CSS** (styling, if included in file or external stylesheet)
* **JavaScript** (client-side behavior, if included)

No backend framework, API layer, or database is currently configured in this repository.

## 🚀 Getting Started
Because this is a static frontend project, setup is very simple.

**Option 1: Open directly in browser**
1.Clone or download the repository.
2.Open InternshipTracker.htm in your browser.

**Option 2: Run with VS Code Live Server (recommended)**
1.Open project folder in VS Code.
2.Install the **Live Server** extension.
3.Right-click InternshipTracker.htm → **Open with Live Server.**

## 💻 Usage
1.Open the app in your browser.
2.Enter internship details (company, role, date, status, notes, etc. based on your UI).
3.Save/manage entries through the interface.
4.Track application progress visually.

## 🔒 Data & Storage Notes
This project is currently frontend-only, so:
* Data may be temporary unless saved in browser storage.
* If localStorage/sessionStorage is used, data stays only on that browser/device.
* There is no cloud sync or user authentication yet.
